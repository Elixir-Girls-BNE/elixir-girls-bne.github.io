Deploying blog to Heroku

Signing up to Heroku is very simple, just head over to Heroku and fill in the form.

Heroku Toolbelt

Once we have signed up, we can download the correct version of the Heroku Toolbelt for our system

Now that we have the Toolbelt installed, let’s create the Heroku application. From our terminal window, in our project directory, run:

heroku create --buildpack "https://github.com/HashNuke/heroku-buildpack-elixir.git"

You should see output similar to below

The name of the Heroku application is the random string after “Creating app...” in the output above (https://fast-journey-73963.herokuapp.com/). This will be unique, so expect to see a different name from “fast-journey-73963”.

The --buildpack option we are passing allows us to specify the Elixir buildpack we want Heroku to use. A buildpack is a convenient way of packaging framework and/or runtime support. In our case it’s installing our application dependencies before we run it.

Next step is to add the Phoenix Static Buildpack. We need to compile static assets for a successful Phoenix deployment. The Phoenix static buildpack can take care of that for us, so let’s add it now inside your project directory in the terminal window.

heroku buildpacks:add https://github.com/gjaldon/heroku-buildpack-phoenix-static.git

Your output from the above command should be similar to below

We also need to add the Heroku Postgres add-on click here to get started

Click the Install Heroku Postgres button

You will be prompted to select the name of your application from the drop down box. In this guide the application is called fast-journey-73963

The plan name will be similar to Hobby Dev - Free, simply select the provision button

Modifying Some Files

We need to create a file in the root of our project called Procfile with the following content

web: MIX_ENV=prod mix phoenix.server

Don't forget to save

Next we need to open the file called prod.exs inside the config folder and replace the line

cache_static_manifest: "priv/static/manifest.json" with cache_static_manifest: "priv/static/manifest.json", secret_key_base: System.get_env("SECRET_KEY_BASE")

Here we’re saying that we want the secret_key_base value to be taken from the environment variable SECRET_KEY_BASE

Next step we need to replace url: [host: "example.com", port: 80], with the name given to you by heroku for your app. Please not this will not be the same as the sample below. Simply replace the host with your app name. Don't forget to ask a mentor if you're stuck

url: [scheme: "https", host: "fast-journey-73963.herokuapp.com", port: 443],
force_ssl: [rewrite_on: [:x_forwarded_proto]],

Finally we need to add/modify the following lines so the file resembles below

config :blog_elixir_girls, BlogElixirGirls.Endpoint, http: [port: {:system, "PORT"}], url: [scheme: "https", host: "fast-journey-73963.herokuapp.com", port: 443], force_ssl: [rewrite_on: [:x_forwarded_proto]], cache_static_manifest: "priv/static/manifest.json", secret_key_base: Map.fetch!(System.get_env(), "SECRET_KEY_BASE") # Do not print debug messages in production config :logger, level: :info config :blog_elixir_girls, BlogElixirGirls.Repo, adapter: Ecto.Adapters.Postgres, username: System.get_env("USER_URL"), url: System.get_env("DATABASE_URL"), pool_size: 20, ssl: true

Find the line import_config "prod.secret.exs", it's located towards the end of the file and delete it. Save your file :)

Almost there..open the file in web/channels/user_socket.ex and replace the line transport :websocket, Phoenix.Transports.WebSocket with

transport :websocket, Phoenix.Transports.WebSocket, timeout: 45_000

Save all files and return to your terminal window. Lets finish this off!

From within your project folder input the following command

heroku addons:create heroku-postgresql:hobby-dev

heroku config:set POOL_SIZE=18

heroku run "POOL_SIZE=2 mix blog_elixir_girls.task"

You need to create the secret key we set in a previous step above. Lets do this now. At the terminal type the following

mix phoenix.gen.secret

A random string will be generated. Now we need to set this in Heroku (remember to use the string your terminal generated and not the one in the example below. In the terminal window enter the following

heroku config:set SECRET_KEY_BASE="yH94rMfqsMpyLW4vkpAELsKLj3o01Shht/hyoTNslPjUo8Xxey3QNeDFdFqn2boN"

Deploy time

Let's commit all our changes, from the terminal window do the following

git add config/prod.exs git add Procfile git add web/channels/user_socket.ex git commit -m 'setting up heroku to use our elixir girls blog app" git push -u heroku master

Deploy

git push heroku master

Hopefully this will work without issue. When done you still need to migrate, so for the final time let's do the following

heroku run "POOL_SIZE=2 mix ecto.migrate"

You are done!!!!
