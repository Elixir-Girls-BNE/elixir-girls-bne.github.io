New Blog

Welcome to Elixir Girls! This guide is for those of you with little or no experience of programming. Together we are going to build a simple blog post (and we might even get to add a comments section too...stay tuned)

Step 1

The first thing we need to do is create a new project and it's best to put this in its own directory. Open your terminal (be sure to as an instructor if unsure) and type the following

mkdir elixir-girls

cd elixir-girls

mix phoenix.new blog_elixir_girls

This command is how we generate a project using the phoenix framework. It will automatically create a lot of folders for you. This will help us put our code in the correct place. Frameworks follow best practice based of years of other developers experience. If you take a look at your terminal you will see lots of output from running that one command

While running the above command you might receive the following message

Fetch and install dependencies? [Yn] Y

If so simply type Y

The output in your terminal will look something like below

Now that the project has started you should cd (stands for change directory) into the new blog_elixir_girls folder. Simply type the command below

cd blog_elixir_girls

If at any stage you are unsure which directory you are in simply type the command below

pwd

This stands for print working directory and will always show you the full path of the directory (or folder) you are in

Open Sublime

Lets open sublime which you might remember is the text editor we installed earlier. Navigate to the directory/folder where you new project is stored (remember the project is called blog_elixir_girls. Just ask a mentor if unsure)

Running the Server

Even though we haven't done much (well in fact we have only typed in one command its a good time to see if we can run our server. From inside your blog_elixir_girls directory type the following command (give it a few mins as it has a bit of work to do). Don't worry if you see some nasty looking errors in red..its just looking for a database and we are about to do that!

mix phoenix.server

Now open a browser window and enter the following url

http://localhost:4000

You should see the Phoenix Framework Welcome Page as below

To terminate the server type ctl-c twice. You should now be back in your terminal window ready to type in more commands

Step 2

Lets use one of the built in Phoenix generators to create our MVC architecture. FYI Phoenix comes with 4 kinds of generators more on this later :) For now simply type the command below into the terminal

mix phoenix.gen.html Post posts title:string body:text

The Phoenix generator has just created a model, view, controller (aswell as some tests and a database connection). We must declare the resource (that's the Post bit in singular and plural for this command to work. We must also specify the type of fields we will need our resource to have. In this case we need a title which is of type String and body which is of type text. String and Text are very similar but text can allow for more data to be input than string. The output will look similar to below

Next you need to simply follow the commands that your terminal window has informed you of. Its really good practice to watch our terminal window for hints about next actions and any issues. Open sublime and navigate to the folder web->router.ex and enter the Post route under scope "/". When done it should look similar to below. Leave the rest of the file as is. Remember to save!

scope "/", BlogElixirGirls do

pipe_through :browser # Use the default browser stack

resources "/posts", PostController

get "/", PageController, :index

end

Return to your terminal window and type in the command below

mix ecto.create

and

mix ecto.migrate

Now we can start our Phoenix Server again (mix phoenix.server). Open a browser window and enter the follow url mix phoenix.server. You should be able to enter a new Post and see a list. Have a play, take some time and remember to keep an eye on the Phoenix Service console (that's running in your terminal window) to see what's going on behind the scenes.

Congrats on the work you've done so far! Next lets add some comments to our blog posts :)
